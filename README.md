# Regular Expressions on a Firewall Log
**Author:** Santiago Yambay
**Course:** Information Systems Engineering — 2nd Semester
**Parallel:** B

## Project Overview
This repository contains a forensic analysis suite utilizing Regular Expressions (Regex) to audit a firewall log dataset. The implementation follows system administration best practices, focusing on pattern matching, capture groups, and I/O stream manipulation using grep and sed.

## Tasks Execution Audit
* **Task 1:** Validates log integrity by excluding headers using anchors.
* **Task 2:** Executes alternation logic to filter rejected traffic.
* **Task 3:** Implements literal dot-escaping to filter network subnets.
* **Task 4:** Audits packet size utilizing quantitative anchors.
* **Task 5:** Reconstructs data streams via sed capture groups.
* **Task 6:** Performs multi-conditional filtering for service audits.
* **Task 7:** Filters chronological time-windows using character-class ranges.
* **Bonus:** Implements negated pattern matching for string validation.
